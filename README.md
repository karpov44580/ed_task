Запуск проекта из репозитория GitHub
  Общие требования:

    Установленный Docker (инструкции по установке смотри ниже).
    Доступ в Интернет для загрузки необходимых образов контейнеров.

Установка Docker
  Для Windows и Mac OS:

    Официальное руководство по установке Docker Desktop доступно здесь:Docker Documentation

    Следуй инструкциям на сайте, чтобы установить последнюю версию Docker Desktop.

  Для Linux:

  Для большинства дистрибутивов Linux установку Docker можно выполнить через пакетный менеджер:

    sudo apt update && sudo apt install docker.io

Убедись, что сервис Docker запущен и доступен:

    sudo systemctl start docker
    sudo systemctl enable docker

Загрузка проекта из GitHub

  Скачай архив проекта или клонируй репозиторий через командную строку:

    git clone https://github.com/karpov44580/ed_task.git
    cd ed_task

После скачивания зайди в папку проекта:

    cd ed_task

Сборка и запуск контейнера

    docker compose build
    docker compose up -d

Это автоматически построит необходимые образы и запустит контейнеры согласно настройкам из файла.

Тестирование приложения

Открой браузер и введи адрес http://localhost или введи в терминале curl http://localhost, чтобы убедиться, что приложение работает корректно.
Результат обоих способов должен показать строку:

    Hello from Effective Mobile!


КАК ЭТО РАБОТАЕТ:

Проект состоит из двух основных компонентов:
    • Backend: простой HTTP-сервер на Python, работающий на порте 8080. Он принимает запросы и отвечает строкой "Hello from Effective Mobile!".
    • Nginx: веб-сервер, выполняющий роль прокси, который принимает внешние запросы на порт 80 и перенаправляет их на внутренний backend-порт 8080.

Схема выглядит так:

Какой-то джентельмен отправляет запрос ←→ Nginx (порт 80) ←→ Backend (порт 8080)

Когда клиент делает запрос на порт 80 nginx, тот прозрачно передает запрос внутреннему backend-у, который обрабатывает запрос и формирует ответ. Затем ответ возвращается клиенту через nginx.
Cхема взаимодействия (ASCII-диаграмма):

                  ┌─────────────┐
                  │   Browser   │
                  └─────────────┘
                        │
                   HTTP Request
                        │
                  ┌─────────────┐

                  ┌─────────────┐
                  │ Nginx Proxy │
                  │ Listening on│

Список использованных технологий:
    • Python 3.10 — язык программирования для реализации backend-приложения.
    • http.server — встроенный модуль Python для создания простого HTTP-сервера.
    • Nginx — обратный прокси-сервер, обрабатывающий запросы клиентов и переправляющий их на backend.
    • Docker — контейнеризация приложения и инфраструктуры.
    • Docker Compose — оркестровка контейнеров, обеспечение взаимодействия и запуска сервисов.
    • Alpine Linux — легкая операционная система, используемая в качестве основы для Docker-образов.
    • Healthcheck — средство проверки работоспособности сервисов в Docker Compose.

Детали реализации:
    1. Backend:
        ◦ Простой HTTP-сервер на Python, отвечающий на GET-запросы по маршруту / текстом "Hello from Effective Mobile!".
        ◦ Контейнер собран с использованием легкого образа Alpine Linux.
    2. Nginx:
        ◦ Выступает в роли обратного прокси, перенаправляя запросы на backend по адресу backend:8080.
        ◦ Содержит минимальную конфигурацию для эффективной работы.
    3. Docker Compose:
        ◦ Управляет созданием и запуском контейнеров.
        ◦ Организует сетевую инфраструктуру для коммуникации между контейнерами.
